# 安全策略

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 0.x（最新） | ✅ 安全补丁 |
| 旧版本 | ❌ 不再维护 |

## 报告漏洞

如果你发现了安全漏洞，请**不要**通过公开的 GitHub Issue 报告。

请通过以下方式私下报告：

1. 使用 GitHub 的 [Security Advisory](https://github.com/aspect-ui/pigeon-ui/security/advisories) 功能创建私密报告。
2. 或发送邮件至维护者邮箱（见项目主页）。

### 报告内容

- 漏洞描述和影响范围
- 复现步骤
- 可能的修复建议（如有）

### 响应时间

- **确认收到**：2 个工作日内
- **初步评估**：5 个工作日内
- **修复发布**：根据严重程度，Critical 级别 7 天内，High 级别 14 天内

## 安全最佳实践

PigeonUI 作为 UI 组件库，遵循以下安全原则：

- 组件内不包含网络请求或外部数据获取逻辑。
- 不使用 `eval()`、`XMLHttpRequest` 等不安全 API。
- 第三方资源（字体、图标）均审计许可证兼容性。
